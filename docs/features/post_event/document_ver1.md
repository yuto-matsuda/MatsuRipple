**イベント投稿機能**についての仕様書です．

# 基本要件
適切なエラーハンドリングやバリデーションを行い，入力内容に不備がある場合はエラーメッセージをフォーム冒頭に表示する．

# 投稿内容
- イベント名
- 開催日
  - 開始日・開始時刻
  - 終了日・終了時刻
- 開催場所
- 説明文 (optional)

# 各フィールドの機能
- 開催日ドラムロールで入力する．ただし，日にちはカレンダーでの選択も可能．
- 開催場所は直接入力を行う他，郵便番号を入力することで自動で補完が可能．
- 開催場所入力後，地図プレビューにピンを表示する．

# UIの特記事項
- カレンダーはデフォルトのものではなく，テーマに併せてデザインすること．
