# MatsuRipple ギャラリー・個人ページ再編 実装案

---

## 背景・目的

現状、写真は以下の3箇所に分散している。

- 写真ギャラリーページ（`/photos`）— 全体の公開写真一覧
- 祭り詳細ページ — 祭りごとの写真
- グループ詳細ページ — グループ共有写真

「写真ギャラリー専用ページ」を廃止し、写真は **個人ページ / グループページ / 祭り詳細** に集約する。
さらに「グループに属さない個人参加者」向けの **個人ページ** を新設する。

---

## 1. 個人ページの新設

### 概要
グループを作らず、個人で祭りに参加したユーザー向けのマイページ。
基本構成は **グループ詳細ページ（GroupDetailPage）と同等** のレイアウト・デザインを踏襲する。

### グループページから流用する要素
| 要素 | 個人ページでの扱い |
|---|---|
| ヘッダー（名前・説明） | ユーザー名を表示 |
| 関連祭り（タグ） | **参加した祭りを自動表示**（手動タグ付けはしない） |
| 写真セクション（一覧 + アップロード + 公開/非公開トグル） | そのまま流用 |

### グループページから除外する要素（個人ページには不要）
- メンバー一覧
- メンバー招待フォーム
- グループ参加申込（一括申込）
- 旅行場所（locations）
- グループ退出ボタン

### ルーティング（確定）
- **`/me`（自分専用ページ）** を新設。他人は閲覧不可（ログインユーザー自身のみ）
- `App.tsx` にルート追加
- 未ログイン時は `/login` へリダイレクト

### 変更・新規ファイル（想定）
| ファイル | 変更内容 |
|---|---|
| `frontend/src/pages/ProfilePage.tsx` | 新規作成（GroupDetailPage をベースに簡略化） |
| `frontend/src/hooks/useProfile.ts` | 個人写真・参加祭りの取得ロジック |
| `frontend/src/App.tsx` | `/me` ルート追加 |
| `frontend/src/components/Navbar.tsx` | 「マイページ」リンク追加 |

### バックエンド
- 既存の `GET /photos/me`（自分の写真）を流用
- 参加した祭りは `participants` を `email` で逆引きするエンドポイントが必要
  - `GET /participants/me` → ログインユーザーが参加した祭り一覧（新規）
  - この一覧をそのまま「関連祭り（自動表示）」として使う

---

## 2. アカウントページに「参加した祭り」セクション追加

### 概要
現在 AccountPage は以下の構成：
- アカウント情報
- 投稿した祭り
- アップロードした写真
- 退会

ここに **「参加した祭り」** セクションを追加する。

### 実装方針
- `GET /participants/me`（上記新規エンドポイント）で取得
- 投稿した祭りカードと同じスタイルのリストで表示
- 各カードから祭り詳細へ遷移

### 変更ファイル
| ファイル | 変更内容 |
|---|---|
| `backend/app/routers/participants.py` | `GET /me` 追加（email 逆引きで参加祭り一覧） |
| `frontend/src/api/participants.ts` | `fetchMyParticipations` 追加 |
| `frontend/src/hooks/useUserHistory.ts` | 参加祭りも取得するよう拡張 |
| `frontend/src/pages/AccountPage.tsx` | 「参加した祭り」セクション追加 |

---

## 3. 写真ギャラリーページの廃止

### 概要
`/photos`（PhotoPage）を削除する。写真は以下に集約：
- 個人ページ（自分の写真）
- グループページ（グループ共有写真）
- 祭り詳細ページ（祭りごとの写真）

### 影響範囲・対応
| ファイル | 変更内容 |
|---|---|
| `frontend/src/App.tsx` | `/photos` ルート削除 |
| `frontend/src/components/Navbar.tsx` | 「写真」リンク削除 |
| `frontend/src/pages/PhotoPage.tsx` | 削除 |
| `frontend/src/hooks/usePhotos.ts` | `festivalId` / `groupId` 指定の利用は維持（全体一覧用途のみ廃止） |

### 写真の閲覧導線（確定）
- 写真ギャラリー廃止後、**公開写真は祭り詳細ページからのみ閲覧可能**とする
- 未ログインユーザー向けの全体写真一覧の導線は設けない

### 注意点
- `usePhotos()`（引数なし＝全体公開写真）を使っている箇所が PhotoPage 以外にないか確認
- アップロード機能（公開/非公開・グループ共有）は個人ページ側へ移管

---

## 4. グループの「関連祭り」も自動表示に変更

### 概要
現状グループ詳細ページの「関連祭り」は **作成者が手動でタグ付け**する仕様。
これを **グループで参加申込した祭りを自動表示**する方式に変更する（個人ページと統一）。

### 実装方針
- `participants` テーブルの `group_id` から、そのグループが参加した祭りを逆引き
- `GET /groups/{id}/festivals` を「参加済み祭りの自動一覧」に変更
- 手動の祭り追加 UI（`POST /groups/{id}/festivals` 等）は廃止

### 影響・確認点
- 既存の `group_festivals` テーブル（手動タグ用）は不要になる
  - データ移行は不要。新方式は `participants.group_id` ベースで算出
- グループ詳細ページの「＋祭りを追加」ボタン・検索フォームを削除

### 変更ファイル（想定）
| ファイル | 変更内容 |
|---|---|
| `backend/app/routers/groups.py` | `GET /{id}/festivals` を participants 逆引きに変更、手動追加/削除エンドポイント廃止 |
| `frontend/src/pages/GroupDetailPage.tsx` | 手動タグ付け UI 削除、自動表示に |
| `frontend/src/api/groups.ts` | `addGroupFestival` / `removeGroupFestival` 廃止 |

## 確定方針（相談済み）

1. **個人ページの URL** → `/me`（自分専用、他人は閲覧不可）
2. **関連祭りの表示** → 個人・グループとも **参加した祭りを自動表示**（手動タグ付けは廃止）
3. **写真の閲覧導線** → **祭り詳細ページからのみ**閲覧可能（全体一覧の導線なし）
